# 插件市场目录说明

## 这份目录是干什么的

这里放的是“插件市场仓库骨架”，不是插件源码仓库。

它只负责给插件市场提供可读的注册表元数据，不负责保存插件运行代码。

## 最小可用结构

```text
marketplace/
  .github/
    ISSUE_TEMPLATE/
    workflows/
  docs/
    contributing/
  market.json
  plugins/
    <plugin_id>/
      entry.json
  schemas/
    entry.schema.json
  scripts/
    validate_plugin_submission.py
    issue_to_entry.py
  samples/
    plugin-submission-issue.md
  templates/
    entry.json.example
```

## 每个目录和文件的职责

### `market.json`

市场根清单。后端会先读取这个文件，再读取 `entry_root` 里的插件条目目录。

当前后端实现要求它至少包含这些字段：

- `market_id`
- `name`
- `owner`
- `repo_url`
- `default_branch`
- `entry_root`
- `trusted_level`

### `plugins/`

真实插件条目目录。

这个目录名必须和 `market.json` 里的 `entry_root` 一致。当前默认就是 `plugins`。

后端同步时只会扫描这个目录下的一级子目录，每个子目录都必须对应一个插件：

```text
plugins/
  weather_forecast/
    entry.json
  calendar_sync/
    entry.json
```

### `plugins/<plugin_id>/entry.json`

单个插件的市场条目元数据。

关键规则只有几条：

- 目录名必须等于 `entry.json` 里的 `plugin_id`
- `source_repo` 必须指向真实插件源码仓库
- `latest_version` 必须能在 `versions` 里找到
- 市场条目只放市场识别所需元数据，不要把整份运行时 manifest 再抄一遍

### `templates/`

模板目录，不参与市场扫描。

模板故意放在 `plugins/` 外面，就是为了避免把示例文件误同步成无效市场条目。

### `.github/`

市场仓库自动化入口。

这里放：

- Issue Form
- GitHub Actions workflow
- 审核状态同步 workflow
- CODEOWNERS
- PR 模板

这套目录负责把“作者提 Issue”变成“机器人 PR + 人工审核”。

如果你想把审批闭环也带过去，记得把 `docs/contributing/repository-settings.md` 一起带上。这个文件不是装饰，它负责说明真实仓库要开哪些 GitHub 开关。

### `scripts/`

市场仓库自动化脚本目录。

当前骨架里已经放了：

- `validate_plugin_submission.py`
- `issue_to_entry.py`

它们负责：

- 解析收录 Issue
- 校验插件仓库
- 生成 `plugins/<plugin_id>/entry.json`

### `schemas/`

条目 schema 目录。

这里放 `entry.json` 的结构定义，方便 CI、脚本和人工审核共用同一套约束。

### `samples/`

本地演练目录。

这里不参与正式市场同步，只给你本地验证自动化脚本用。

## 复制到未来市场仓库时怎么用

1. 先把 `market.json` 放到未来仓库根目录。
2. 保留 `plugins/` 作为市场条目根目录。
3. 每新增一个插件，就新建 `plugins/<plugin_id>/entry.json`。
4. 把 `templates/entry.json.example` 复制成真实条目后，再按插件真实信息改掉占位值。
5. 如果你要启用自动收录流程，把 `.github/`、`scripts/`、`schemas/` 一起复制到目标市场仓库。
6. 复制完后，再按 `docs/contributing/repository-settings.md` 打开目标仓库的 GitHub 权限和分支保护。

## 默认市场地址在哪里配

后端默认市场地址在这里定义：

- `apps/api-server/app/core/config.py`

对应代码常量：

- `OFFICIAL_PLUGIN_MARKETPLACE_REPO_URL`
- `OFFICIAL_PLUGIN_MARKETPLACE_BRANCH`
- `OFFICIAL_PLUGIN_MARKETPLACE_ENTRY_ROOT`

这里现在就是写死在代码里的，不再允许通过环境变量覆盖官方市场地址。

如果以后官方市场仓库地址真的要变，就直接改这里的代码常量，不要再走部署配置。

当前固定地址是：

- `https://github.com/jingyi0605/familyclaw-marketplace.git`
- 分支：`main`
